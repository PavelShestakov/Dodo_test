# Dodo_test
Прототип для детекции событий по одному столику на видео

Небольшой прототип для детекции событий по одному столику на видео:
- стол пустой
- стол занят
- подход к столу

Проект сделан по тестовому заданию: выбрать один видимый столик, отслеживать события только для него, сохранять события в `Pandas DataFrame`, считать среднее время между освобождением стола и следующим подходом человека, а также сохранить видео с визуализацией. :contentReference[oaicite:0]{index=0}

## Запуск

Установить зависимости:

bash
```pip install -r requirements.txt```


Запустить скрипт:

```python main.py --video "путь_до_видео.mp4"```

## После запуска:

На первом кадре вручную выбрать нужный столик через cv2.selectROI.
Дождаться окончания обработки.

## На выходе сохраняются:

- output.mp4 — видео с визуализацией
- events.csv — таблица событий
- Выбранное видео и столик
Видео: Видео 1
Выбранный столик: Правый верхний столик, возле прохода, в центре зала.

## Логика детекции:

В проекте используется YOLOv8n для поиска людей в кадре. На первом кадре пользователь вручную выбирает ROI столика. Далее для каждого кадра:

- детектируются люди,
- проверяется, попадает ли центр бокса человека внутрь ROI столика,

Cостояние стола обновляется по простой логике:
Пуст — человека в зоне нет,
Занят — человек в зоне есть,
Подход к столу — человек появился после периода пустоты.

Чтобы убрать случайные ложные переключения, используются пороги по количеству секунд:

MIN_PRESENT_SECONDS = 3
MIN_ABSENT_SECONDS = 4

Все события сразу записываются в DataFrame, а затем сохраняются в events.csv. Средняя задержка считается как время между событием table_empty и следующим событием approach.
